# Friends Feature Implementation Plan

**Goal**: Add friends and view their movie/TV history when public

## Overview

- **Mode**: Cloud mode only (requires Supabase Auth)
- **Data visibility**: Public feed (all watch history visible to friends)
- **Connection model**: Mutual approval (friend requests)
- **Scope**: View friends' movies/TV shows with same filtering/sorting as own content

## Database Schema Changes

Create `supabase/migrations/0002_friends.sql`:

```sql
-- Friend request table with mutual approval workflow
create table if not exists friend_requests (
  id bigint generated by default as identity primary key,
  requester_id uuid not null references auth.users(id) on delete cascade,
  recipient_id uuid not null references auth.users(id) on delete cascade,
  status text not null default 'pending' check (status in ('pending', 'accepted', 'rejected')),
  created_at timestamptz default now(),
  updated_at timestamptz default now(),
  
  unique(requester_id, recipient_id)
);

-- Indexes for fast lookups
create index if not exists friend_requests_recipient_idx on friend_requests (recipient_id, status);
create index if not exists friend_requests_requester_idx on friend_requests (requester_id, status);

-- RLS: Users can see requests involving them
alter table friend_requests enable row level security;

create policy "view own requests" on friend_requests for select 
  using (auth.uid() = requester_id or auth.uid() = recipient_id);

create policy "create requests" on friend_requests for insert 
  with check (auth.uid() = requester_id);

create policy "update requests" on friend_requests for update 
  using (auth.uid() = recipient_id and status = 'pending');

-- User profiles
create table if not exists profiles (
  id uuid primary key references auth.users(id) on delete cascade,
  display_name text,
  bio text,
  avatar_url text,
  created_at timestamptz default now()
);

alter table profiles enable row level security;

create policy "view all profiles" on profiles for select using (true);
create policy "update own profile" on profiles for update using (auth.uid() = id);
create policy "insert own profile" on profiles for insert with check (auth.uid() = id);

-- Trigger to update updated_at
create or replace function update_updated_at()
returns trigger as $$
begin
  new.updated_at = now();
  return new;
end;
$$ language plpgsql;

create trigger friend_requests_updated_at 
  before update on friend_requests 
  for each row execute procedure update_updated_at();
```

## Backend Changes

None required (cloud mode uses direct Supabase calls)

## Frontend Implementation

### 1. New API Layer

Create `frontend/src/api/friendsApi.js` with functions for:
- `sendRequest(recipientId)` - Send friend request
- `getPendingRequests()` - Get incoming/outgoing pending requests
- `acceptRequest(requestId)` - Accept friend request
- `rejectRequest(requestId)` - Reject friend request
- `getFriends()` - Get all accepted friends
- `getFriendMovies(friendId, params)` - Get friend's movies/TV with filters
- `removeFriend(friendId)` - Remove friend
- `getMyProfile()` - Get own profile
- `updateMyProfile(updates)` - Update own profile

### 2. New Context

Create `frontend/src/utils/FriendsContext.jsx`:
- State: friends, pendingRequests, myProfile, loading, error
- Methods: refreshFriends, sendRequest, acceptRequest, rejectRequest, removeFriend, updateProfile
- Add to `main.jsx` providers

### 3. New Pages

**FriendsPage** (`frontend/src/pages/FriendsPage.jsx`):
- Tab navigation: Friends / Requests / Find Friends
- Friends tab: List friends with "View Profile" button
- Requests tab: Accept/reject incoming, cancel outgoing
- Find Friends tab: Search by email, add friend

**FriendProfilePage** (`/friends/:friendId`):
- Profile header (avatar, name, bio)
- Movie/TV tabs with filtering/sorting
- Read-only view of friend's content

### 4. UI Components

Create `frontend/src/components/friends/`:
- `FriendCard.jsx` - Display friend in list
- `FriendRequestCard.jsx` - Pending request with actions
- `ProfileEditor.jsx` - Edit display name, bio, avatar
- `AddFriendModal.jsx` - Search and send request

### 5. Navigation Updates

Add to `App.jsx`:
```javascript
<Route path="/friends" element={<FriendsPage />} />
<Route path="/friends/:friendId" element={<FriendProfilePage />} />
```

Add "Friends" button/link to main navigation

### 6. Profile Auto-creation

Update `AuthGate.jsx` to create profile on signup

## Implementation Order

1. Database migration (`supabase/migrations/0002_friends.sql`)
2. API layer (`frontend/src/api/friendsApi.js`)
3. Context (`frontend/src/utils/FriendsContext.jsx`) + add to main.jsx
4. UI components (FriendCard, FriendRequestCard, etc.)
5. FriendProfilePage (reuse MovieCard/SeriesCard, filters)
6. FriendsPage with tabs
7. Navigation updates
8. Profile auto-creation in AuthGate

## Key Design Decisions

- **Read-only friend content**: Friends can view but not edit
- **Cloud mode only**: Requires Supabase for multi-user
- **Public feed**: All watch history visible to friends
- **Mutual approval**: Friend requests must be accepted
- **Reuse components**: MovieCard, filters, sorting work for friend data
